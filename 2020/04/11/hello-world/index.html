搭建个人blog，step by step，hexo + githubPage => hello world !

Created2020-04-11|Updated2023-11-20|tecStudy
|Post View:|Comments:

一、概述

  • Hexo: 一个基于Node.js的静态博客框架，使用Markdown等标记语言来撰写文章，可以生成静态的HTML网页文件。Hexo支持多种主题和插件，具有快速、简单、易于使用等特点，是一个非常受欢迎的静态博客框架。
  • GitHub Pages: 是GitHub提供的一项服务，它允许用户创建和托管静态网站，而无需自己搭建服务器或购买域名等。用户可以将自己的静态网页文件（如HTML、CSS、JavaScript等）存储在GitHub仓库中，然后通过GitHub Pages服务将这些文件托管在互联网上，从而实现了建立个人博客、展示作品集等目的。
  • hexo + githubPage的组合即是利用Hexo框架来撰写文章生成需要的静态文件，将这些文件交由githubPage管理，使他们能够供互联网上的其他人访问来实现个人博客。

二、环境准备

  • 安装node.js,概述已经说过Hexo是基于NodeJS编写的，所以需要安装NodeJS和npm工具来作为Hexo的环境
    这个环境想必大家都有，如果没有的话可以参考这里安装node.js。
  • 2、需要有一个GitHub账号，没有的话到官网申请一个，不多赘述。

3、将博客托管在github page
Part1，准备githubPage

  • 登录github账号，创建一个仓库用于存放你发布的博客

  • 设置仓库名，格式必须为**<用户名>.github.io**，然后点击Create repository，完成创建

  • 创建完成后打开你刚刚创建的仓库，点击setting一栏找到pages设置,可以看见创建的githubpage的域名（图内绿框代表你的用户名）

  • 至此，我们已经有了一个允许互联网上其他人访问的窗口（刚刚得到的域名）
Part2，使本地的博客可以发布到githubPage上
  • 配置SSH key，即是将本地的github账户与互联网上的github账户匹配，从而能确保提交内容能被顺利更新到对应仓库
  • cd ~/.ssh，如果没报错或者提示什么的说明就是以前生成过的（以前使用git工具时），直接使用 cat ~/.ssh/id_rsa.pub 命令就是可以查看本机上的 SSH key 了。
  • 如这一步发生报错或是提示可参考这里来完成生成生成本地公钥

  • ok，到这一步我们手里应该已经拿到了本地公钥，接下来到我们的github账号，将这个公钥配置到我们的github账户，完成本地与github账号的连接


  • 至此，本地与github的连接也完成了，验证一下连接是否畅通⑧
  • 设置成功后，即可不需要账号密码和github互动辣
  • 回到hexo，此时，虽然本地可以和github畅通了，但是我们发布hexo博客并不是简单使用git将本地的博客文件整个提交到github，而是收到我们的发布命令后，hexo会选择性的将有必要上传的文件上传，大概就是经过hexo渲染后需要展示给博客访问者的内容，而本地的框架配置文件等等则不会上传。
  • 为了达到上一条描述额效果，我们需要安装部署插件hexo-deployer-git

  • 稍等片刻，访问 https://你的用户名.github.io 就会看到你发布的博客辣!
